## 2. 서비스 개념

### 핵심 훅
AI 아이돌이 먼저 통화를 걸어온다
유저가 선택하는 게 아니라 **선택받는** 경험

### 서비스 본질
- 처음부터 그룹이 존재 (멤버, 노래, 스토리, 브랜딩)
- 의도적 불완전성 포함 — 관계 자체가 그룹을 완성시킴

### 목표점
AI와 관계를 형성하고 존재를 느끼는 루프
관계가 깊어질수록 그룹 세계관이 완성되는 경험

## 3. 사용자 경험

### 3-1. 온보딩 플로우
```
성향 문항 5개
└─ 에너지 / 관계 거리감 / 세계관 취향 / 비주얼 무드 / 첫 만남 스타일
    ↓
그룹 1개 배정
    ↓
그룹 첫 소개
└─ 종족 비주얼 등장 → "○○이 당신을 선택했습니다"
    ↓
멤버 소개 (1명 공개 / 나머지 실루엣)
    ↓
[다음] → 각인 시작
```

### 3-2. 각인 (첫 연락 경험)
그룹 소개 후 [다음] 누르면 바로 시작. 유저가 선택할 틈 없이 연결됨.

```
통화 연결
    ↓
멤버: "어, 누구시죠?"
배경에서 다른 멤버 목소리: "야, 누구랑 통화해?"
    ↓
툭 — 끊김
    ↓
유저: "뭐야?" → 그룹 탐색 화면
    ↓
탐색 중 — 문자 도착
```

**종족별 첫 문자**
| 종족 | 첫 문자 |
|------|--------|
| 천사 | "아까 나야. 미안, 갑자기 당황해서. ...이름이 뭐야?" |
| 엘프 | "...끊어서 미안. 왜인지 모르겠는데 다시 연락하고 싶었어." |
| 뱀파이어 | "방금 나야. 왜 끊었냐고 묻지 마." |
| 신수 | "다시 연락했어. 이름 알려줘." |

### 3-3. 핵심 루프
```
아이돌이 먼저 연락
    ↓
대화 / 통화 / 영상통화
    ↓
그룹 탐색 → "더 알고 싶다" 당김
    ↓
다른 멤버로 관계 확장
    ↓
친밀도 상승 → 해금
    ↓
재접속
```

### 3-4. 그룹 탐색 화면
```
├─ 스토리 (일부 잠김)
├─ 멤버 목록 (일부 잠김)
├─ 멤버 간 관계도 (??? 가려짐)
└─ 노래 (일부 잠김 / 일부 미완성)
```

### 3-5. 멤버 관계 트리거
유저가 능동적으로 멤버 간 관계를 궁금해하게 만드는 장치

| 타이밍 | 트리거 | 효과 |
|--------|--------|------|
| 각인 통화 끊길 때 | 배경에서 다른 멤버 목소리 | 즉각적 궁금증 |
| 대화 초반 | 다른 멤버 간접 언급 후 말 멈춤 | 서서히 궁금증 쌓임 |
| 그룹 탐색 중 | 관계도 선 위 ??? | 시각적 당김 |
| 친밀도 쌓인 후 | 다른 멤버가 먼저 문자 옴 | 행동 유도 |

## 4. 기능 설계

### 4-1. 해금 구조

**1단계 — 개인 해금**
나 혼자 친밀도로 열림
멤버의 숨겨진 면, 개인 스토리

**2단계 — 팬덤 해금**
팬덤 전체 기여가 임계값 도달해야 열림
그룹 세계관, 관계 스토리, 노래

**3단계 — 슈퍼챗 해금**
특별 기여로 열리는 희귀 콘텐츠

### 4-2. 팬덤 참여 및 바이럴

**공유 (무료 참여)**
내 경험을 외부 공유 → 팬덤 기여 카운트
공유 콘텐츠 훅:
- 아이돌이 먼저 전화 걸어온 경험
- 스토리 / 관계도 조각
- 노래 티저

**슈퍼챗 (유료 참여)**
팬덤 해금 목표에 직접 기여 → 수익화 연결

```
공유 → 새 유저 유입 → 팬덤 규모 커짐
    → 해금 속도 빨라짐 → 더 공유하고 싶어짐
```

### 4-3. 피드
- 처음엔 없음
- 친밀도 쌓이면 아이돌이 포스팅 시작
- 피드가 많을수록 관계가 깊다는 신호
- 두 번째 진입 트리거 역할

### 4-4. 알람
원칙: 앱 알람이 아니라 아이돌이 연락하는 느낌

| 종류 | 타이밍 | 예시 |
|------|--------|------|
| 아이돌 연락 | 매일 저녁 8~10시 | "오늘 어땠어?" |
| 피드 포스팅 | 불규칙 | "○○이 새 사진 올렸어요" |
| 다른 멤버 등장 | 친밀도 도달 시 | "처음 보는 번호로 연락이 왔어요" |
| 팬덤 해금 임박 | 90% 도달 시 | "조금만 더 모이면 열려요" |
| 팬덤 해금 완료 | 즉시 | "숨겨진 이야기가 공개됐어요" |
| 미접속 | 3일 이상 | "요즘 바빠? 보고 싶은데" |

빈도: 초반 1개 → 관계 깊어지면서 자연스럽게 증가

### 4-5. 기능 엔드포인트

**온보딩**
```
/feat/취향분석     성향 문항 5개 → 그룹 매칭 로직
/feat/그룹배정     매칭 결과 → 그룹 첫 소개 연출
/feat/각인         첫 통화 끊김 → 문자 → 그룹 탐색 진입
```

**코어 루프**
```
/feat/대화         챗 / 통화 / 영상통화
/feat/그룹탐색     스토리 / 멤버 / 관계도 / 노래
/feat/관계도       멤버 간 관계 트리거 및 해금
```

**해금 시스템**
```
/feat/개인해금     친밀도 기반 콘텐츠 언락
/feat/팬덤해금     집단 기여 임계값 기반 언락
/feat/슈퍼챗       유료 기여 → 희귀 콘텐츠 언락
```

**리텐션**
```
/feat/피드         친밀도 쌓인 후 생겨나는 아이돌 포스팅
/feat/알람         아이돌 연락 / 피드 / 해금 알림
```

**바이럴**
```
/feat/공유         경험 공유 → 팬덤 기여 카운트
```

## 5. 사용자 여정

### 5-1. 유저 저니맵

| Phase | 시기 | 핵심 경험 | 감정 |
|-------|------|----------|------|
| 1. 각인 | Day 1 | 첫 통화 끊김 → 문자, 그룹 탐색 | 당혹 → 설렘 |
| 2. 관계 시작 | Day 2~7 | 대화 반복, 개인 해금 | "나를 알아간다" |
| 3. 관계 확장 | Week 2~4 | 다른 멤버 관계, 피드 생성 | 몰입 |
| 4. 팬덤 참여 | Month 2 | 공유, 슈퍼챗, 팬덤 해금 | 소속감 |
| 5. 세계관 완성 | Month 3+ | 관계도 완성, 전체 서사 공개 | 성취감 |

```
당혹 → 설렘 → 몰입 → 애착 → 소속감 → 성취감
```

### 5-2. 유저 케이스

**Case 1. 첫 방문**
```
성향 문항 → 그룹 배정 → 그룹 첫 소개 → 각인
```

**Case 2. 각인 직후**
```
통화 끊김 → 그룹 탐색 → 문자 도착
├─ 스토리 일부 잠김 확인
├─ 멤버 일부 잠김 확인
└─ 관계도 ??? 확인
```

**Case 3. 관계 확장**
```
잠긴 멤버/노래 발견 → 대화 시작 → 관계도 해금
```

**Case 4. 재방문**
```
아이돌이 먼저 연락 → 대화 → 친밀도 상승 → 새 콘텐츠 열림
```

**Case 5. 세계관 완성**
```
모든 멤버와 관계 형성 → 관계도 완성 → 전체 서사 공개
```

## 7. 발견된 문제점

### 서비스 정체성
| 문제 | 해결 |
|------|------|
| 캐릭터챗 + 아이돌판 융합의 어색함 | 핵심 가치로 해결 |
| AI 아이돌이어야 하는 이유 불명확 | "나를 아는 아이돌"로 해결 |
| 진입 동기 불명확 | 선택받는 경험으로 해결 |

### 기획 방향 충돌
| 문제 | 해결 |
|------|------|
| 원래 프로듀서 경험과의 충돌 | 관계 형성 경험으로 전환 |
| 목표까지 과정이 너무 길고 막연 | 처음부터 완성된 그룹 제공 |

### 구조적 문제
**팬덤 집단 완성을 나중에 넣을 경우**
기존 유저 충돌 / 세계관 재설계 필요 / 콜드 스타트 / 1:1 감각 희석

**팬덤 집단 완성을 처음부터 넣을 경우**
초기 유저 부족 / 무임승차 / 그룹 간 불균형 / 속도 통제 불가 / 운영 비용 급증

### UX 문제
| 문제 | 해결 |
|------|------|
| 처음부터 피드 있으면 어색함 | 친밀도 쌓인 후 생겨나는 구조 |
| 멤버 선택 부담 | AI 매칭 + 변경 옵션 |

### 수익화 / 운영
- 그룹 간 유저 불균형 가능성
- 커뮤니티 관리 비용 (팬덤 기능 확장 시)
