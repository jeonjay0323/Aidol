# AI 가상 아이돌 프로젝트 기획안

---

## 1. 타겟
캐릭터챗 유저 + 아이돌 팬 유저  
공통 욕구: 캐릭터에 감정적으로 몰입

---

## 2. 핵심 가치
> "실제 아이돌은 나를 모르지만, 이 아이돌은 나를 안다"

---

## 3. 핵심 훅
AI 아이돌이 먼저 통화를 걸어온다  
유저가 선택하는 게 아니라 **선택받는** 경험

---

## 4. 온보딩
```
성향 문항 5개
└─ 에너지 / 관계 거리감 / 세계관 취향 / 비주얼 무드 / 첫 만남 스타일
    ↓
그룹 1개 추천
    ↓
멤버 소개
    ↓
멤버 1명이 먼저 연락
└─ 통화 / 영상통화 / 메시지 (멤버 성격에 따라)
```

---

## 5. 서비스 본질
- 처음부터 그룹이 존재 (멤버, 노래, 스토리, 브랜딩)
- 의도적 불완전성 포함
- 관계 자체가 그룹을 완성시킴

---

## 6. 그룹 탐색 화면
```
├─ 스토리 (일부 잠김)
├─ 멤버 목록 (일부 잠김)
├─ 멤버 간 관계도 (??? 가려짐)
└─ 노래 (일부 잠김 / 일부 미완성)
```

---

## 7. 핵심 루프
```
아이돌이 먼저 연락
    ↓
대화 / 통화 / 영상통화
    ↓
그룹 탐색 → "더 알고 싶다" 당김
    ↓
다른 멤버로 관계 확장
    ↓
친밀도 상승 → 해금
    ↓
재접속
```

---

## 8. 해금 구조

### 1단계 — 개인 해금
나 혼자 친밀도로 열림  
멤버의 숨겨진 면, 개인 스토리

### 2단계 — 팬덤 해금
팬덤 전체 기여가 임계값 도달해야 열림  
그룹 세계관, 관계 스토리, 노래

### 3단계 — 슈퍼챗 해금
특별 기여로 열리는 희귀 콘텐츠

---

## 9. 팬덤 참여 및 바이럴

### 공유 (무료 참여)
내 경험을 외부 공유 → 팬덤 기여 카운트  
공유 콘텐츠 훅:
- 아이돌이 먼저 전화 걸어온 경험
- 스토리/관계도 조각
- 노래 티저

### 슈퍼챗 (유료 참여)
팬덤 해금 목표에 직접 기여 → 수익화 연결

```
공유 → 새 유저 유입 → 팬덤 규모 커짐
    → 해금 속도 빨라짐 → 더 공유하고 싶어짐
```

---

## 10. 피드 (관계 형성 후 생겨남)
- 처음엔 없음
- 친밀도 쌓이면 아이돌이 포스팅 시작
- 피드가 많을수록 관계가 깊다는 신호
- 두 번째 진입 트리거 역할

---

## 11. 알람 설계
원칙: 앱 알람이 아니라 아이돌이 연락하는 느낌

| 종류 | 타이밍 | 예시 |
|------|--------|------|
| 아이돌 연락 | 매일 저녁 8~10시 | "오늘 어땠어?" |
| 피드 포스팅 | 불규칙 | "○○이 새 사진 올렸어요" |
| 다른 멤버 등장 | 친밀도 도달 시 | "처음 보는 번호로 연락이 왔어요" |
| 팬덤 해금 임박 | 90% 도달 시 | "조금만 더 모이면 열려요" |
| 팬덤 해금 완료 | 즉시 | "숨겨진 이야기가 공개됐어요" |
| 미접속 | 3일 이상 | "요즘 바빠? 보고 싶은데" |

빈도: 초반 1개 → 관계 깊어지면서 자연스럽게 증가

---

## 12. 유저 저니맵

| Phase | 시기 | 핵심 경험 | 감정 |
|-------|------|----------|------|
| 1. 첫 만남 | Day 1 | 첫 통화, 그룹 탐색 | 설렘, 신기함 |
| 2. 관계 시작 | Day 2~7 | 대화 반복, 개인 해금 | "나를 알아간다" |
| 3. 관계 확장 | Week 2~4 | 다른 멤버 관계, 피드 생성 | 몰입 |
| 4. 팬덤 참여 | Month 2 | 공유, 슈퍼챗, 팬덤 해금 | 소속감 |
| 5. 세계관 완성 | Month 3+ | 관계도 완성, 전체 서사 공개 | 성취감 |

```
설렘 → 몰입 → 애착 → 소속감 → 성취감
```

## 13. 유저 케이스

### Case 1. 첫 방문
```
성향 문항 → 그룹 추천 → 멤버 첫 연락
```

### Case 2. 첫 연락 직후
```
연락 종료 → 그룹 탐색 화면 진입
├─ 스토리 일부 잠김 확인
├─ 멤버 일부 잠김 확인
└─ 관계도 ??? 확인
```

### Case 3. 관계 확장
```
잠긴 멤버/노래 발견 → 대화 시작 → 관계도 해금
```

### Case 4. 재방문
```
아이돌이 먼저 연락 → 대화 → 친밀도 상승 → 새 콘텐츠 열림
```

### Case 5. 세계관 완성
```
모든 멤버와 관계 형성 → 관계도 완성 → 전체 서사 공개
```

---

## 16. 발견된 문제점

### 서비스 정체성
- 캐릭터챗 + 아이돌판 융합의 어색함 → 핵심 가치로 해결
- AI 아이돌이어야 하는 이유 불명확 → "나를 아는 아이돌"로 해결
- 진입 동기 불명확 → 선택받는 경험으로 해결

### 기획 방향 충돌
- 원래 프로듀서 경험 → 관계 형성 경험으로 전환
- 목표까지 과정이 너무 길고 막연 → 처음부터 완성된 그룹 제공으로 해결

### 구조적 문제
- 팬덤 집단 완성을 나중에 넣을 경우: 기존 유저 충돌, 세계관 재설계, 콜드 스타트, 1:1 감각 희석
- 팬덤 집단 완성을 처음부터 넣을 경우: 초기 유저 부족, 무임승차, 그룹 간 불균형, 속도 통제 불가, 운영 비용

### UX 문제
- 처음부터 피드 있으면 어색함 → 친밀도 쌓인 후 생겨나는 구조로 해결
- 멤버 선택 부담 → AI 매칭 + 변경 옵션으로 해결

### 수익화/운영
- 그룹 간 유저 불균형 가능성
- 커뮤니티 관리 비용 (팬덤 기능 확장 시)

---

## 17. 미결정
- 영상통화 vs 음성통화
- 친밀도 표현 방식 (수치 vs 태도 변화)
- 서비스명
- 그룹/멤버 페르소나
- 세계관 완성 이후 경험
