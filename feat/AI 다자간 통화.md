# AI 다자간 통화 기획안

## 목표

여러 AI 연습생 캐릭터끼리 설정된 상황에 맞게 자유롭게 대화하는 콘텐츠 시스템 구현

---

## 기술 스택

| 역할 | 선택 | 이유 |
|------|------|------|
| 대화 생성 | Gemini 2.5 Flash | 빠르고 자연스러운 한국어 구어체 생성 |
| TTS | Gemini 2.5 Flash Preview TTS | 캐릭터별 남성 목소리 적용 |
| 백엔드 | Python FastAPI + WebSocket | 실시간 스트리밍 및 ACK 동기화 |
| 프론트엔드 | 바닐라 JS | 분할 화면 UI, 오디오 재생 |

---

## 아키텍처

```
브라우저 (duo.html)
    ↓ WebSocket
FastAPI 서버 (server.py /ws/duo)
    ↓ 1단계: 대화 텍스트 일괄 생성 (Gemini 2.5 Flash)
    ↓ 2단계: TTS 파이프라인 (Gemini TTS, 1줄씩 선행 생성)
브라우저 오디오 재생 (AudioContext 24kHz PCM)
```

### 핵심 설계 결정

**1단계 텍스트 일괄 생성**
- 매 줄마다 LLM 호출하면 대화 텀이 길어짐
- 전체 대화(24줄)를 LLM 1회 호출로 생성 → 초기 생성 후 빠르게 진행
- 생성 즉시 클라이언트에 "대본 생성 중..." 피드백 전송

**TTS 파이프라인 (1줄 선행 생성)**
- 현재 줄 재생 중 → 다음 줄 TTS 미리 생성
- 줄 사이 대기시간 최소화
- `asyncio.create_task`로 백그라운드 병렬 처리

**ACK 동기화**
- 서버: 한 줄 전송 → 클라이언트 ACK 대기 → 다음 줄 전송
- 클라이언트: 오디오 재생 완료 후 ACK 전송
- 재생 타이밍에 맞춰 자막·화자 하이라이트 동기화

**분할 화면 UI**
- 말하는 캐릭터가 확장(`flex: 1.35`), 나머지 어두워짐
- 하단 자막으로 현재 대사 표시
- Simli 립싱크 미사용 (무료 플랜 동시 연결 1개 제한)

## 대화 생성 프롬프트 전략

- 두 캐릭터의 성격/소개 포함
- 상황 설정 (기본값: "연습 끝나고 숙소에서 샤워 먼저 하겠다고 서로 안 비켜주는 상황")
- 2024년 한국 10~20대 구어체 강제
  - 감탄사로 문장 시작 (아, 야, 어, 진짜 등)
  - `~거든?`, `~잖아`, `~냐고` 반말체 어미
  - 한 대사 최대 20자 이내
  - 이모지·ㅋㅋ 금지, 소리 나는 그대로만
